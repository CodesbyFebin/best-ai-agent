# Our Methodology: How We Test & Rank AI Agents

Transparent explanation of our 42-point evaluation methodology for AI agent reviews and rankings.

## 42-Point Scoring Matrix

Every AI agent is evaluated across 8 core pillars. Each pillar contains multiple sub-criteria, totaling 42 evaluation points.

### 1. Capability (15%, 6 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| Task coverage | 2% | Breadth of tasks the agent can complete |
| Model quality | 2% | Underlying model performance and reasoning |
| Reasoning | 1% | Multi-step reasoning and logic |
| Tool use | 1% | Ability to use external tools and APIs |

### 2. Ease of Use (15%, 6 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| Setup | 2% | Time and complexity to get started |
| UI/UX | 2% | Interface quality and intuitiveness |
| Onboarding | 1% | Tutorials, guides, and learning curve |
| Hinglish support | 1% | Indian language support |

### 3. Features (10%, 4 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| Integrations | 2.5% | Third-party integrations available |
| Automations | 1% | Built-in automation capabilities |
| Plugins | 0.5% | Extensibility and plugin ecosystem |

### 4. Documentation (10%, 4 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| API docs | 2.5% | API documentation quality |
| Examples | 1% | Code examples and tutorials |
| INR examples | 0.5% | India-specific examples |

### 5. Pricing (10%, 4 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| Transparency | 2.5% | Clear and upfront pricing |
| Value | 1% | Value for money |
| INR pricing | 0.5% | India-specific pricing |

### 6. Reliability (15%, 6 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| Uptime | 2% | Service availability and stability |
| Latency | 2% | Response time and speed |
| Error handling | 1% | Graceful error recovery |
| India servers | 1% | India-based infrastructure |

### 7. Support (10%, 4 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| Response time | 2.5% | Support response time |
| Channels | 1% | Available support channels |
| Local hours | 0.5% | India-friendly support hours |

### 8. Security & Compliance (15%, 6 points)

| Sub-criteria | Weight | What We Test |
|--------------|--------|--------------|
| Encryption | 2% | Data encryption in transit and at rest |
| Access control | 2% | User access management |
| Audit logs | 1% | Activity logging and monitoring |
| DPDP readiness | 1% | DPDP Act compliance |

## Testing Protocol

### Duration
- Minimum 40 hours of hands-on testing per agent
- Tests conducted over 2-4 weeks for accuracy

### Environments
- **Sandbox:** Isolated testing environment for security
- **Production:** Real-world testing with actual workflows
- **India-specific:** Testing from Indian IP addresses and payment methods

### Data Collection
- Feature verification against official documentation
- Pricing verification on vendor websites
- Benchmark results from official leaderboxes
- User community feedback analysis

## Ranking Process

1. **Data collection:** Gather all evaluation data
2. **Scoring:** Score each sub-criteria 1-10
3. **Weighting:** Apply category weights
4. **Review:** Dual review by editorial team
5. **Verification:** Fact-checking and source verification
6. **Publication:** Publish with methodology disclosure

## Update Frequency

- **Reviews:** Updated when major product changes occur
- **Rankings:** Re-evaluated quarterly
- **Benchmarks:** Updated as new results are published
- **Methodology:** Reviewed semi-annually

### Implementation checklist
1. Define the target workflow, owner, user, input data, and expected output.
2. Estimate monthly cost in INR, including tax treatment and possible overages.
3. Check whether the vendor can provide suitable invoices, procurement terms, and admin controls.
4. Review DPDP Act 2023 implications if personal data is processed.
5. Test English, Hindi, Hinglish, and regional-language examples where relevant.
6. Validate WhatsApp, UPI, Razorpay, CRM, helpdesk, cloud, or database integrations with the exact workflow.
7. Pilot with a small team and compare results against the existing manual process.
8. Document escalation rules, monitoring, rollback steps, and review cadence.
