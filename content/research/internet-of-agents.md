## Agent Collaboration Patterns
 
### Sequential Workflow Pattern
 Agents execute in sequence:
 ```
 User -> AgentA -> AgentB -> AgentC -> Result
 Each agent completes before next begins
 Simple to debug and trace
 Single point of failure
```
 
### Parallel Decomposition Pattern
 Main agent splits work:
 ```
 Main Agent -> [AgentA, AgentB, AgentC] simultaneously
 Results aggregated after completion
 Faster overall execution
 Higher resource usage
```
 
### Hierarchical Coordination Pattern
 Supervisor manages team:
 ```
 Supervisor Agent -> Specialist Agents
                    -> Review Agents
                    -> Quality Check
Central coordination with quality gates
Clear responsibility chains
Potential bottleneck at supervisor
```
 
### Peer-to-Peer Pattern
 Agents negotiate directly:
 ```
 AgentA <-> AgentB <-> AgentC
 Dynamic collaboration
 Complex coordination
 Resilient to failures
